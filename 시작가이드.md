# 생활도구함 시작 가이드

코딩을 몰라도 됩니다. 순서대로만 따라오세요.
막히는 단계가 있으면 이 프로젝트 대화방에서 클로드에게 "○단계에서 막혔어"라고 물어보세요.

> 💡 **한 줄 요약**: 사이트가 이상하게 보이거나(디자인 깨짐, 페이지 404) 하면 열에 아홉은 "GitHub에 폴더 구조가 깨진 채로 올라감" 문제입니다. 2단계 ②번의 주의사항부터 확인하세요.

---

## 이 폴더의 구조

```
tool-site/
├── index.html          ← 홈 (도구 목록)
├── assets/style.css    ← 공통 디자인 (건드릴 일 거의 없음)
└── tools/
    └── bmi.html        ← 예시 도구. 새 도구는 여기에 파일로 추가
```

새 도구 추가 = ① tools 폴더에 html 파일 넣기 + ② index.html에 카드 한 개 붙여넣기.
이게 전부입니다.

---

## 1단계. 계정 만들기 (약 20분, 최초 1회)

**꼭 크롬(Chrome) 브라우저로 진행하세요.** 다른 브라우저는 폴더 업로드가 잘 안 될 수 있습니다.

1. **GitHub** (github.com) 가입
   - Sign up → 이메일 → 비밀번호 → 사용자 이름(영문) 순서로 입력
   - 이메일로 온 인증 코드 입력하면 완료. 이후 뜨는 설문은 아래 **Skip** 눌러 건너뛰어도 됨
2. **Vercel** (vercel.com) 가입
   - Sign Up → 계정 유형은 **Hobby**(개인, 무료) 선택
   - **Continue with GitHub** 클릭 → 방금 만든 GitHub 계정으로 로그인 → **Authorize** 클릭
   - 보안 관련 화면(2FA 설정 권유)이 뜨면 지금은 **"Skip securing my account"**로 넘어가도 됩니다.
     (사이트가 자리잡고 광고가 붙기 시작하면 그때 다시 설정하는 걸 권장)

## 2단계. 사이트 올리기 (최초 1회)

**① GitHub에 저장소 만들기**
1. GitHub 로그인 상태에서 오른쪽 위 **+** → **New repository**
2. Repository name: `tool-site` 입력, **Public** 선택 확인
3. **Create repository** 클릭

**② 파일 업로드 — 가장 실수하기 쉬운 단계, 아래 주의사항 꼭 확인**
1. 저장소 화면에서 **uploading an existing file** 링크 클릭
2. 컴퓨터에서 압축을 미리 풀어둔 `tool-site` 폴더를 **연 다음**, 안에 있는
   `index.html`, `assets` 폴더, `tools` 폴더, `시작가이드.md`, `프롬프트모음.md`를
   **전부 선택**(Ctrl 누르고 하나씩 클릭)해서 브라우저의 점선 박스 안으로 드래그

   > ⚠️ **주의 1**: `tool-site` 폴더 자체를 통째로 끌지 말고, 폴더를 열어서 **그 안의 내용물**을 끌어야 합니다. 폴더째 올리면 경로가 한 단계 깊어져서 사이트가 깨집니다.
   >
   > ⚠️ **주의 2**: `assets`, `tools`처럼 폴더가 있는 항목을 드래그했는데도, 가끔 폴더 구조가 무시되고 안의 파일만 낱개로(예: `bmi.html`, `style.css`가 최상위에 바로) 올라가는 경우가 있습니다. 이렇게 되면 사이트에서 **CSS가 안 입혀지고 도구 페이지가 404 에러**가 납니다.

3. 업로드 후, 파일 목록에 폴더 아이콘(📁)으로 `assets`, `tools`가 보이는지 **반드시 확인**하세요.
   - ✅ 정상: `assets/`, `tools/`가 폴더 아이콘으로 보이고, 그 안에 각각 `style.css`, `bmi.html`이 있음
   - ❌ 문제 발생: `bmi.html`, `style.css`가 폴더 없이 최상위에 파일로 그냥 보임
   - 문제가 있다면 → 잘못 올라간 파일들을 각각 열어서 휴지통 아이콘으로 삭제(Commit changes) 후, ②번을 다시 시도
4. 확인 후 아래로 스크롤해서 **Commit changes** 클릭

**③ Vercel로 배포**
1. vercel.com → **Add New... → Project**
2. "Import Git Repository" 목록에서 `tool-site` 옆의 **Import** 클릭
   (목록에 안 보이면 처음 한 번 GitHub 앱 설치/권한 허용 화면이 뜰 수 있음 → `tool-site` 저장소를 선택해서 **Install** 진행)
3. Project Name은 그대로 두고, Application Preset은 **Other** 그대로 (건드릴 필요 없음)
4. Build and Output Settings, Environment Variables는 펼치지 않고 그대로 둠
5. **Deploy** 클릭 → 1~2분 후 축하 화면이 뜨면 성공
6. **Visit** 버튼(또는 대시보드 상단 링크)을 눌러 실제 주소(`tool-site-xxxx.vercel.app`)로 접속

**④ 최종 확인 (이 3가지를 꼭 눈으로 확인)**
1. 연두빛 배경에 "생활도구함" 로고와 디자인이 정상적으로 보이는지
2. BMI 카드를 클릭했을 때 404 에러 없이 도구 페이지로 이동하는지
3. 키·몸무게 입력 후 "계산하기"를 눌렀을 때 결과가 나오는지

문제가 있다면 → GitHub 저장소 화면을 캡처해서 클로드에게 보여주면 원인(대부분 폴더 구조 문제)을 바로 찾아줍니다.

**⑤ 이후 업데이트는 Vercel 재배포 버튼 없이 자동으로!**
GitHub 저장소에 새 파일을 커밋(Commit)하면, Vercel이 자동으로 감지해서 1~2분 안에 사이트를 다시 배포합니다. 5단계(새 도구 추가)부터는 이 자동 배포를 그대로 활용하면 됩니다.

## 3단계. 도메인 연결 (선택이지만 강력 추천)

- 애드센스 승인과 신뢰도에 도움이 됩니다. 연 1~2만원 수준.
- 가비아, 후이즈, Cloudflare 등에서 `.com` 또는 `.kr` 구매
- Vercel 프로젝트 → Settings → Domains 에서 연결 (안내가 화면에 나옴)

## 4단계. 검색 등록 (트래픽의 시작)

한국은 네이버 유입이 큽니다. 둘 다 꼭 하세요.
1. **구글 서치콘솔** (search.google.com/search-console) — 사이트 등록 + 사이트맵 제출
2. **네이버 서치어드바이저** (searchadvisor.naver.com) — 사이트 등록 + 수집 요청

## 5단계. 새 도구 추가하는 루틴 (매주 1~2개)

1. 이 프로젝트 대화방에서 클로드에게 요청
   → `프롬프트모음.md` 파일에 있는 프롬프트를 복사해 쓰세요
2. 받은 html 파일을 GitHub 저장소의 `tools` 폴더에 업로드
   (저장소에서 tools 폴더 클릭 → Add file → Upload files)
3. `index.html`을 열어(연필 아이콘) 카드 템플릿 주석을 복사해 새 카드 추가 → Commit
4. Vercel이 자동으로 다시 배포해 줍니다 (1~2분). 끝!

## 6단계. 애드센스 신청 (사이트에 도구 5개 + 글이 충분해진 뒤)

1. adsense.google.com 에서 사이트 등록
2. 심사 팁: 도구마다 붙어 있는 설명 글이 심사 통과의 핵심입니다.
   빈약한 페이지가 많으면 거절될 수 있으니, 도구 5~8개를 채우고 신청하세요.
3. 승인되면 각 페이지의 `ad-slot` 자리에 광고 코드를 붙여넣습니다.
   (이때도 클로드에게 "광고 코드 어디에 넣어?"라고 물어보면 됩니다)
4. **개인정보처리방침 페이지**가 필요합니다 — 클로드에게 만들어 달라고 하세요.

## 현실적인 기대치

- 1~3개월: 수익 0원이 정상. 도구와 글을 쌓는 시기
- 3~6개월: 검색 유입 시작, 애드센스 승인, 첫 수익(월 몇천 원~몇만 원)
- 6개월~: 잘 크는 카테고리를 발견하면 그쪽으로 집중 → 월 수십만 원 목표

포기하는 사람 대부분이 3개월 안에 그만둡니다.
반대로 말하면, 꾸준히만 하면 경쟁자가 계속 사라져 주는 게임입니다.
